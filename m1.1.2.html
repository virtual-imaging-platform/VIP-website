Ontology in OWL / OntoSpec (M1.1.2)

This is the first release of the ontology in the OWL format. We also added the OntoSpec file which describes the concepts in a semi-formal representation.


The ontology is, for the moment, focused on the modelization of the simulation models. It is broken down into several OWL files, the main file to open is vip-model.owl. It is based on the DOLCE foundational ontology, and on the NeurLOG ontologies. The more convenient way to browse the ontology is to use the Protégé editor : http://protege.stanford.edu/ (v4.*)

This version contains :

  • the description of the models according to different semantic axes
  • the description of the content of the model (layer/layer-part/objects)
  • the description of the physical parameters
  • the anatomical objects extracted from the FMA according to the list of terms identified from the models and the physical parameters
  • the foreign-body / radiopharmaceutical / contrast agent extracted from the RadLex terminology.

The next important steps are :

  • identify the relation with the IAMF format
  • feed the model with pathological / pathology (need reflexion : diseases, pathology, pathological object)
  • develop the next ontologies (simulator, simulated data)

The ontology evolves constantly according to your frequent meeting, discussion and choice. The version is consequently subject to change. Do not hesitate to give us feedback or if you experience trouble trying to open it.
